### crystallize — Crystal Management (store / find)

Persist structured engineering artifacts into the crystal memory system and search for existing crystals. Call `store` at the completion of each skill layer (after user approval). Call `find` to locate existing crystals by type, module, layer, or semantic similarity.

**Parameters:**
- `command` (string, optional, default `"store"`): Which operation to perform.
  - `"store"` — Store a new crystal.
  - `"find"`  — Search for existing crystals.

---

#### store (default)

- `crystal_type` (string, required): Type of crystal. Must match the current layer:
  - L1 → `ArchCrystal` (architecture_summary, tech_stack, core_flow)
  - L2 → `ModMap` (modules[], dependencies{})
  - L3 → `ContractCrystal` (signature, preconditions[], postconditions[], constraints[])
  - L4 → `LogicCrystal` (algorithm_steps[], boundary_handling{})
  - L5 → `LogicCrystal` (algorithm_steps[] — rigorous pseudocode, typically skipped)
  - L6 → `SkeletonCrystal` (code_skeleton, language)
  - L7 → `ImplCrystal` (code, tests[], language)
  - L8 → `TraceCrystal` (symptom, root_cause, fix)
  - L3_record / L7_record → `ModuleRecord` (module snapshot for archiving, see below)
  - (any) → `ExperienceCrystal` (cross-project reusable experience, not tied to a specific layer)
- `module` (string, required): Module name this crystal belongs to.
- `name` (string, required): Human-readable name (function name, decision title).
- `content` (string, required): JSON string with structured content matching the crystal type.

**When to use:**
- After the user approves output at any skill layer (L1, L2, L3, L6)
- After completing implementation (L7) for each function
- When a bug is diagnosed (L8) — store a TraceCrystal

**ModuleRecord content schema:**
- `record_type`: `"L3_snapshot"` or `"L7_snapshot"`
- L3_snapshot fields: `contract_signature`, `preconditions`, `postconditions`
- L7_snapshot fields: `impl_files`, `test_results`, `algorithm_summary`
- Optional: `renegotiation_notes`, `parent_contract_id`

**ExperienceCrystal content schema:**
- `title` (required, string): Short experience title
- `summary` (required, string): 1-2 sentence description of the experience
- `problem` (optional, string): Problem encountered
- `solution` (optional, string): Solution applied
- `reference_values` (optional, dict): Dimension→insight mapping, e.g. `{"debug": "...", "architecture": "..."}`
- `tags` (optional, list): Search keywords in Chinese and English
- `source_project` (optional, string): Origin project for traceability

ExperienceCrystals are automatically marked as persistent and embedded into the vector DB for cross-project search. They survive project archiving and are available to all future projects.

**When NOT to use:**
- Before the user has approved the current layer's output
- For temporary or speculative content

---

#### find

- `crystal_type` (string, optional): Filter by crystal type (e.g. `"ContractCrystal"`, `"TraceCrystal"`).
- `module` (string, optional): Filter by module name.
- `layer` (string, optional): Filter by layer (e.g. `"L3"`, `"L7"`).
- `query` (string, optional): Semantic search query. When provided, searches ContractCrystal signatures, TraceCrystal symptoms, and ExperienceCrystal persistent entries using vector/keyword similarity. Results include a relevance score.
- `limit` (integer, optional, default 20): Maximum number of results.

**Two search modes:**
1. **Structured filter** (no `query`): Returns all active crystals matching the given filters, sorted by vitality (most reused first). Useful for listing all contracts in a module, all crystals at a layer, etc.
2. **Vector similarity** (with `query`): Searches ContractCrystal signatures and TraceCrystal symptoms semantically. Results are ranked by cosine similarity + vitality bonus.

**When to use:**
- Before writing a new contract — check if a similar one already exists
- Before debugging — search for related traces
- When entering a module — list its existing crystals for context
- When unsure what crystals exist for the current project

Note: `ExperienceCrystal` can be created via `crystallize store` directly, or generated by the `archive_project` tool which synthesizes ModuleRecord + TraceCrystal data into cross-project reusable experience.
